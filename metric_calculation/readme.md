# YOLO K-Fold Evaluation

## Model Directory Structure

Expected trained model structure:

```text
runs/
   synthetic_kfold/
       fold_1/
           weights/
               best.pt

       fold_2/
           weights/
               best.pt

       fold_3/
       fold_4/
       fold_5/
```

Each fold must contain:

```text
weights/best.pt
```

## Dataset Format

Expected YOLO dataset structure:

```text
dataset/
│
├── images/
│   ├── img_001.jpg
│   ├── img_002.jpg
│   └── ...
│
├── labels/
│   ├── img_001.txt
│   ├── img_002.txt
│   └── ...
│
└── dataset.yaml
```

Example `dataset.yaml`:

```yaml
path: dataset

train: images
val: images
test: images

names:
  0: gun
```

## Basic Usage

```bash
python evaluate_kfold.py \
    --models runs/synthetic_kfold \
    --data dataset/test_black_gun/data.yaml \
    --output result/synthetic_kfold
```

## Example with Custom Parameters

```bash
python evaluate_kfold.py \
    --models runs/synthetic_kfold \
    --data dataset/test_black_gun/data.yaml \
    --output result/synthetic_kfold \
    --imgsz 1088 \
    --batch 16 \
    --conf 0.25 \
    --iou 0.7 \
    --device 0
```


## CLI Arguments

| Argument | Description                     |
| -------- | ------------------------------- |
| --models | Path to k-fold models directory |
| --data   | Dataset YAML file               |
| --output | Output directory                |
| --imgsz  | Image size                      |
| --batch  | Batch size                      |
| --conf   | Confidence threshold            |
| --iou    | IoU threshold                   |
| --device | CUDA device                     |

---

## Default Parameters

```text
Image Size: 640
Batch Size: 16
Confidence: 0.25
IoU: 0.7
Device: 0
```

# YOLO Benchmark Summary

## Model Directory Structure

Expected experiment structure (Ultralytics training output):

```text
runs/
   segment/
       exp1/
           results.csv

       exp2/
           results.csv

       exp3/
           results.csv
```

Or recursively:

```text
runs/
   segment/
       exp1/
           results.csv
       nested/
           exp2/
               results.csv
```

Each experiment must contain:

```text
results.csv
```

This file is automatically generated during training by Ultralytics YOLO.

## Metrics Used

The script extracts the following metrics from `results.csv`:

* Precision
* Recall
* F1 Score (computed)
* mAP50
* mAP50-95
* Fitness (weighted score)

### Fitness Formula

Fitness is computed as a weighted combination:

```
fitness = w1 * precision + w2 * recall + w3 * mAP50 + w4 * mAP50-95
```

Default weights:

```text
[0.0, 0.0, 0.1, 0.9]
```

## Basic Usage

```bash
python benchmark.py \
    --root runs/segment
```

## Example with Custom Parameters

```bash
python benchmark.py \
    --root runs \
    --recursive \
    --weights 0 0 0.3 0.7 \
    --output results/benchmark_summary
```

## CLI Arguments

| Argument    | Description                                          |
| ----------- | ---------------------------------------------------- |
| --root      | Root directory containing experiment runs            |
| --weights   | Fitness weights (precision, recall, mAP50, mAP50-95) |
| --recursive | Search for `results.csv` recursively                 |
| --output    | Output file prefix (CSV & Excel)                     |

## Default Parameters

```text
Weights: [0.0, 0.0, 0.1, 0.9]
Recursive Search: Disabled
Output: benchmark_summary
```

## Output

The script generates:

```text
benchmark_summary.csv
benchmark_summary.xlsx
```

Example output:

```text
model   epoch  precision  recall  f1_score  fitness  mAP50  mAP50-95
exp3    87     0.9123     0.8841  0.8980    0.7452   0.9210 0.7315
exp1    92     0.9055     0.8702  0.8875    0.7321   0.9102 0.7204
exp2    78     0.8991     0.8600  0.8791    0.7210   0.9050 0.7098
```

Models are automatically sorted by **best fitness score**.
